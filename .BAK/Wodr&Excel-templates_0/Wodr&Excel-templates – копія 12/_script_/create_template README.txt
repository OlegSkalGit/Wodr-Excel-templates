========================================================================
ДЕТАЛЬНИЙ ПОСІБНИК: СИСТЕМА ГЕНЕРАЦІЇ ШАБЛОНІВ ТА КОНФІГІВ
========================================================================
Цей посібник описує роботу трьох скриптів, призначених для автоматизації 
створення шаблонів Word/Excel та вилучення з них унікальних даних.

Всі три скрипти базуються на одному й тому ж інтелектуальному рушії 
порівняння текстів та збереження форматів, але відрізняються ступенем 
автоматизації та підходами до роботи з великими обсягами даних.

------------------------------------------------------------------------
1. create_template.py (Базове попарне порівняння)
------------------------------------------------------------------------
ПРИЗНАЧЕННЯ: 
Порівнює рівно ДВА документи (наприклад, старий і новий) і знаходить 
усі текстові розбіжності між ними.

ЯК ПРАЦЮЄ:
Скрипт поабзацно (для Word) або поклітинково (для Excel) перевіряє 
обидва файли. Якщо знайдено розбіжність (різні імена, дати, суми), 
він виділяє цю ділянку як змінну (наприклад, {{ field_1 }}). 
На виході він генерує готовий шаблон з підставленими змінними та 
Excel-конфіг, у якому вже записані дані з цих двох файлів (2 рядки).

ВИКОРИСТАННЯ:
python _script_\create_template.py <файл_1> <файл_2>

ПРИКЛАД:
У вас є два схожі договори: "Договір_Іванов.docx" та "Договір_Петров.docx".
Команда: python _script_\create_template.py "Договір_Іванов.docx" "Договір_Петров.docx"

Результат (з'явиться у папці першого файлу):
- Договір_Іванов_template.docx (Де замість прізвищ буде стояти {{ field_1 }})
- Договір_Іванов_config.xlsx (Таблиця з двома рядками: для Іванова та Петрова)
- Договір_Іванов_run_all.bat (Файл для швидкого запуску main.py)

------------------------------------------------------------------------
2. create_template_package.py (Пакетне порівняння однієї групи)
------------------------------------------------------------------------
ПРИЗНАЧЕННЯ:
Дозволяє взяти ОДИН файл-зразок і порівняти його з УСІМА аналогічними 
файлами у вказаній папці. Ідеально для масового збору даних.

ЯК ПРАЦЮЄ:
Ви вказуєте конкретний файл-зразок та папку. Скрипт рекурсивно сканує 
папку та знаходить всі файли з таким самим розширенням (.docx або .xlsx). 
Далі він одночасно порівнює всі знайдені документи зі зразком. 
Будь-який фрагмент, що відрізняється хоча б в одному файлі, стає змінною.
Унікальні дані з усіх файлів записуються до конфігурації без дублювання 
(однакові рядки автоматично ігноруються).

ФІЛЬТРИ БЕЗПЕКИ:
- Файли, що мають слово "template" у назві, ігноруються.
- Файли, всередині яких вже є синтаксис змінних (наприклад, {{ name }}), 
  ігноруються як "вже готові шаблони".

ВИКОРИСТАННЯ:
python _script_\create_template_package.py <файл_зразок> <папка_з_файлами>

ПРИКЛАД:
У вас є 50 договорів у папці "Архів 2023".
Команда: python _script_\create_template_package.py "Архів 2023\Договір_1.docx" "Архів 2023"

Результат (з'явиться у папці файлу-зразка):
- Договір_1_template.docx (Універсальний шаблон для всіх 50 файлів)
- Договір_1_config.xlsx (Таблиця з 50 рядками унікальних даних клієнтів)
- Договір_1_run_all.bat

------------------------------------------------------------------------
3. create_template_package_full_auto.py (Повний автопілот)
------------------------------------------------------------------------
ПРИЗНАЧЕННЯ:
Сканує всю папку, самостійно розбиває всі файли на групи за їхньою 
структурою і для кожної групи генерує свій шаблон та СПІЛЬНИЙ конфіг.

ЯК ПРАЦЮЄ:
Це найпотужніший інструмент для "сліпого" аналізу архівів.
1. Знаходить всі .docx та .xlsx файли у папці та підпапках.
2. Розбиває їх на "структурні групи". Для Word це файли з однаковою 
   кількістю абзаців та таблиць. Для Excel — з однаковою кількістю аркушів.
3. Кожну групу аналізує (як попередній скрипт) і створює для неї свій шаблон.
4. ІНТЕЛЕКТУАЛЬНЕ ІМ'Я ФАЙЛУ: Скрипт перевіряє оригінальні назви документів. 
   Якщо в назві зустрічається значення знайденої змінної (наприклад, прізвище), 
   він створить динамічний шаблон імені файлу (напр. "Акт_{{ field_1 }}.docx") 
   і запише його в комірку A2.
5. Записує дані всіх груп в єдиний файл Auto_Config.xlsx, створюючи для 
   кожної групи свій окремий аркуш.

ВИКОРИСТАННЯ:
python _script_\create_template_package_full_auto.py <папка_з_документами>

ПРИКЛАД:
У папці "Бухгалтерія" лежать впереміш Акти, Договори та Рахунки.
Команда: python _script_\create_template_package_full_auto.py "Бухгалтерія"

Результат (з'явиться у папці, звідки запущено скрипт):
- Auto_Config.xlsx (Мульти-конфіг: матиме 3 аркуші: Акти, Договори, Рахунки).
- Акт_template.docx (Згенерований шаблон акту)
- Договір_template.docx (Згенерований шаблон договору)
- Рахунок_template.xlsx (Згенерований шаблон рахунку)
- Auto_Run_All.bat (Один скрипт для масової генерації абсолютно всіх документів)

========================================================================
ВНУТРІШНІ ОСОБЛИВОСТІ (Спільні для всієї системи)
========================================================================

А. РОЗУМНИЙ ПОШУК ЗМІННИХ (Smart Tokenization)
- Скрипти шукають розбіжності по цілих словах і числах (токенах).
- Дрібні розбіжності (пробіли, коми) згладжуються і приєднуються до змінної.
- Наприклад, якщо в одному файлі "Іванов Іван", а в іншому "Петров Петро", 
  вся фраза перетвориться на єдиний тег {{ field_1 }}, а не на розірвані шматки.

Б. ЗБЕРЕЖЕННЯ ФОРМАТУВАННЯ
- У Word: Бездоганно зберігається вирівнювання тексту (по центру, по краях) 
  у таблицях і абзацах. Зберігається жирний шрифт, курсив, підкреслення, 
  розмір та тип шрифту (через перебір runs).
- В Excel: Зберігаються всі формули (вони ігноруються під час пошуку змінних), 
  а числові дані залишаються числами.

В. ІГНОРУВАННЯ СМІТТЯ
Якщо скрипти знаходять файли, які не вдається прочитати, або які мають 
неочікувану структуру, вони не зупиняють роботу (не "падають" з помилкою), 
а продовжують обробляти наступні коректні файли.
