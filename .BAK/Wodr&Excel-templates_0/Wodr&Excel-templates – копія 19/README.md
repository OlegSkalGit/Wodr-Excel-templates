# 🚀 СУПЕР-СИСТЕМА АВТОМАТИЗАЦІЇ ДОКУМЕНТІВ: _templates_machine_

Ласкаво просимо до **_templates_machine_** — універсального та інтелектуального "комбайну" для автоматизації роботи з документами **Word (.docx)** та **Excel (.xlsx)**. 

Ця система повністю об'єднує весь цикл обробки документів в одному файлі: від автоматичного аналізу архівів документів та створення розумних шаблонів із виділенням змінних, до масової генерації сотень нових документів на основі таблиць конфігурації.

## ⚡ Швидкий старт: Налаштування вашого ПК

Для того щоб система працювала на вашому комп'ютері, потрібно встановити Python та необхідні бібліотеки. Ми підготували автоматичний інсталятор, який зробить усе за один клік!

### 📥 Покрокова інсталяція:
1. Перейдіть до папки [setup/](setup/).
2. Запустіть файл **[INSTALL.bat](setup/INSTALL.bat)** (двічі клацніть лівою кнопкою миші).
3. **Що відбудеться автоматично**:
   - Інсталятор запустить тиху установку офіційного **[Python 3.14.5](setup/python-3.14.5-amd64.exe)**, автоматично додасть його до системних шляхів (PATH) та налаштує для всіх користувачів.
   - Створить ізольоване віртуальне оточення `.venv` у корені проекту.
   - Завантажить та встановить усі необхідні бібліотеки з **[requirements.txt](setup/requirements.txt)** (`openpyxl` та `docxtpl`).

*Усе готово! Ніяких ручних налаштувань консолі чи встановлення бібліотек.*

## 🎮 Режими роботи системи

Скрипт автоматично розпізнає потрібний режим на основі переданих йому аргументів.

### 🔍 Частина 1: Створення шаблонів та конфігів (Аналіз)

Скрипт порівнює документи, виявляє місця, де текст відрізняється, замінює ці розбіжності на змінні `{{field_1}}`, `{{field_2}}` тощо, зберігаючи оригінальне форматування (курсив, підкреслення, пробіли, межі підпису), та генерує Excel-таблицю налаштувань.

#### 1️⃣ Режим "Повний автопілот"
Аналізує всю папку, групує схожі файли за структурою, створює для кожної групи один шаблон і загальний конфіг.
```bash
python _templates_machine_.py <назва_папки>
```
*Приклад:* `python _templates_machine_.py "Архів"`

#### 2️⃣ Режим "Пакетний аналіз"
Бере один файл-зразок і порівнює його з усіма схожими файлами в папці для пошуку змінних.
```bash
python _templates_machine_.py <зразок> <папка>
```
*Приклад:* `python _templates_machine_.py "було.docx" "Архів"`

#### 3️⃣ Режим "Попарне порівняння"
Швидко порівнює два конкретних файли та генерує шаблон з конфігом.
```bash
python _templates_machine_.py <файл_1> <файл_2>
```
*Приклад:* `python _templates_machine_.py "Договір_Київ.docx" "Договір_Львів.docx"`

---

### 🖨️ Частина 2: Масова генерація документів (Синтез)

Зчитує дані зі створеного Excel-конфігу та миттєво створює готові документи з підставленими значеннями.

```bash
python _templates_machine_.py <файл_конфігу.xlsx> [аркуш] [рядок]
```

#### Варіанти фільтрації при запуску:
* **Згенерувати все з конфігу**:
  ```bash
  python _templates_machine_.py "Auto_Config.xlsx"
  ```
* **Генерувати тільки один конкретний аркуш (за назвою або індексом)**:
  ```bash
  python _templates_machine_.py "Auto_Config.xlsx" "Договори"
  # Або за номером аркуша (наприклад, 1-й аркуш):
  python _templates_machine_.py "Auto_Config.xlsx" 1
  ```
* **Генерувати конкретний рядок з певного аркуша (наприклад, 5-й рядок)**:
  ```bash
  python _templates_machine_.py "Auto_Config.xlsx" "Договори" 5
  ```
* **Генерувати конкретний рядок з усіх аркушів одразу**:
  ```bash
  python _templates_machine_.py "Auto_Config.xlsx" all 5
  ```

## 💎 Унікальні переваги та інтелектуальні функції

### 🛡️ Авто-лікування кирилиці у Windows CMD (Anti-Mojibake)
Під час запуску скрипту через класичні `.bat` файли або командний рядок Windows кириличні аргументи (назви аркушів типу `"Аркуш1"`) часто перетворюються на нечитабельний набір символів (`РђСЂРєСѓС€1`). **Скрипт автоматично детектує такі викривлення та відновлює оригінальний текст**, забезпечуючи безперебійний запуск!

### 🎯 Розумне збереження пробілів та форматування
Пробіли на межах змінних (особливо при підкресленнях) зберігаються до останнього символу. Вирівнювання, курсив, жирний шрифт та межі підкреслень залишаються рівно такими, якими були до автоматичної заміни.

### 🧩 Безпечна робота зі злитими комірками таблиць
Скрипт містить унікальний XML-трекінг комірок (`cell._tc`). Він захищає від помилок `NoneType` при аналізі об'єднаних зон та виключає повторне сканування злитих комірок, що запобігає запису назв сусідніх змінних замість значень.

### 📂 Відтворення оригінальної структури папок
Якщо ваші оригінальні файли були вкладені в підпапки (наприклад, `in/`), скрипт автоматично врахує це, пропише шлях у конфігурацію, а під час генерації **сам створить відсутні підпапки** і розкладе готові документи на свої місця.
