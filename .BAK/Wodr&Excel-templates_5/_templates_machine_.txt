========================================================================
ДЕТАЛЬНИЙ ПОСІБНИК: СУПЕР-СИСТЕМА "_templates_machine_"
========================================================================
Цей посібник описує роботу універсального "комбайну" _templates_machine_.py, 
який повністю закриває весь цикл автоматизації документів: від розумного 
аналізу архівів та створення шаблонів до масової генерації готових 
файлів на основі створеної конфігурації.

Уся логіка знаходиться в ОДНОМУ файлі. Скрипт автоматично розпізнає, 
яку задачу ви хочете вирішити, залежно від переданих параметрів.

------------------------------------------------------------------------
ЧАСТИНА 1: СТВОРЕННЯ ШАБЛОНІВ ТА КОНФІГІВ (Аналіз)
------------------------------------------------------------------------
В усіх режимах працює інтелектуальний алгоритм порівняння текстів та 
збереження оригінального форматування (стилів, пробілів, меж підкреслення). 
Об'єднані комірки та складна розмітка обробляються автоматично і безпечно.

РЕЖИМ 1: ПОВНИЙ АВТОПІЛОТ (Тільки Папка)
Сканує вказану папку, розбиває файли на структурні групи, для кожної генерує 
свій шаблон і об'єднує все в єдиний мульти-конфіг `Auto_Config.xlsx`.
Використання: python _templates_machine_.py <папка>
Приклад:      python _templates_machine_.py "Архів"

РЕЖИМ 2: ПАКЕТНИЙ АНАЛІЗ (Файл-зразок + Папка)
Бере один файл-зразок і порівнює його з усіма подібними файлами в папці.
Використання: python _templates_machine_.py <зразок> <папка>
Приклад:      python _templates_machine_.py "Договір1.docx" "Архів"

РЕЖИМ 3: ПОПАРНЕ ПОРІВНЯННЯ (2 файли)
Знаходить розбіжності між двома конкретними файлами і створює шаблон.
Використання: python _templates_machine_.py <файл_1> <файл_2>
Приклад:      python _templates_machine_.py "Файл1.docx" "Файл2.docx"

------------------------------------------------------------------------
ЧАСТИНА 2: ГЕНЕРАЦІЯ ГОТОВИХ ДОКУМЕНТІВ (Синтез)
------------------------------------------------------------------------
Цей режим зчитує створені Excel-конфіги, завантажує змінні й автоматично 
генерує готові Word (.docx) чи Excel (.xlsx) документи.

РЕЖИМ 4: ЗАПУСК ГЕНЕРАЦІЇ (Конфіг + фільтри аркушів та рядків)
Використання: 
python _templates_machine_.py <файл_конфігу.xlsx> [аркуш] [рядок]

АРГУМЕНТИ ГЕНЕРАЦІЇ:
1. <файл_конфігу.xlsx> (Обов'язковий) — шлях до файлу налаштувань.
2. [аркуш] (Опціональний, за замовчуванням "all") — вказує, які аркуші обробляти.
   Можливі варіанти:
   - "all" — згенерувати документи з усіх аркушів конфігу.
   - Назва аркуша (наприклад, "Договори", "Аркуш1") — регістр літер та пробіли 
     навколо назви ігноруються для запобігання помилкам. Повністю підтримується кирилиця.
   - Номер аркуша (наприклад, 1, 2) — 1-базовий порядковий індекс аркуша в книзі.
3. [рядок] (Опціональний, за замовчуванням "all") — вказує, які рядки з даними обробляти.
   Можливі варіанти:
   - "all" — згенерувати документи для всіх рядків з даними (починаючи з 5-го рядка).
   - Номер рядка (наприклад, 5, 6, 12) — згенерувати документ тільки для конкретного 
     рядка з даними (за номером рядка в Excel).

ПРИКЛАДИ ЗАПУСКУ:
1. Згенерувати все з усіх аркушів конфігу:
   python _templates_machine_.py "Auto_Config.xlsx"

2. Згенерувати документи тільки з аркуша "Договори" (всі рядки):
   python _templates_machine_.py "Auto_Config.xlsx" "Договори"

3. Згенерувати документи з першого аркуша конфігу (всі рядки):
   python _templates_machine_.py "Auto_Config.xlsx" 1

4. Згенерувати документи з аркуша "Аркуш1" тільки для 5-го рядка:
   python _templates_machine_.py "Auto_Config.xlsx" "Аркуш1" 5

5. Згенерувати документ з першого аркуша конфігу тільки для 6-го рядка:
   python _templates_machine_.py "Auto_Config.xlsx" 1 6

6. Згенерувати 5-й рядок з усіх аркушів конфігу:
   python _templates_machine_.py "Auto_Config.xlsx" all 5

========================================================================
ВНУТРІШНІ ОСОБЛИВОСТІ ТА АВТО-ЛІКУВАННЯ
========================================================================

А. АВТО-ЛІКУВАННЯ КИРИЛИЦІ В CMD (MOJIBAKE)
При запуску скрипту з консолі Windows (CMD) чи через .bat файли кириличні назви 
аркушів часто передаються у викривленому кодуванні (Mojibake типу "РђСЂРєСѓС€").
Скрипт автоматично виявляє такі викривлення, перекодовує та лікує їх назад 
у правильний Юнікод ("Аркуш1"), гарантуючи 100% запуск за будь-яких налаштувань консолі.

Б. СУПЕР-ТОЛЕРАНТНІСТЬ ДО РЕГІСТРУ ТА ПРОБІЛІВ
При порівнянні назв аркушів скрипт ігнорує розбіжності у великих/малих літерах, 
а також випадкові зайві пробіли на початку чи в кінці назви аркуша.

В. РОЗУМНЕ ФОРМУВАННЯ ІМЕН ТА ПІДПРИЄМСТВО ПАПОК
Під час аналізу скрипт вибудовує шаблон вихідного імені в комірці A2. 
Якщо вхідні файли знаходились у підкаталогах (наприклад, "in/"), цей шлях 
автоматично прописується в A2. Під час генерації скрипт сам створить ці підпапки!

Г. ВБУДОВАНІ СИСТЕМНІ ЗМІННІ
Ви можете використовувати динамічні дати у назвах файлів та в тексті:
{{ YYYY }}, {{ MM }}, {{ DD }}, {{ hh }}, {{ mm }}, {{ ss }} 
(Рік, Місяць, День, Години, Хвилини, Секунди на момент запуску генерації).

Д. БЕЗПЕЧНА ОБРОБКА ЗЛИТИХ КОМІРОК
Таблиці з об'єднаними комірками аналізуються без дублювання змінних чи помилок 
зчитування порожніх елементів (NoneType).
