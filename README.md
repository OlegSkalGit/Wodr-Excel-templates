# 🚀 СУПЕР-СИСТЕМА АВТОМАТИЗАЦІЇ ДОКУМЕНТІВ: _templates_machine_

Ласкаво просимо до **_templates_machine_** — універсального та інтелектуального "комбайну" для автоматизації роботи з документами **Word (.docx)** та **Excel (.xlsx)**. 

Ця система повністю об'єднує весь цикл обробки документів в одному місці: від автоматичного аналізу архівів документів та створення розумних шаблонів із виділенням змінних, до масової генерації сотень нових документів на основі таблиць конфігурації.

---

## 🎯 Швидкий покроковий алгоритм роботи (Workflow)

Для швидкого старту та розуміння логіки роботи системи виконайте наступні кроки:

1.  **Збір документів-зразків**: Зберіть кілька заповнених прикладів одного документа (наприклад, 2-3 раніше створені договори чи акти) в окрему папку.
2.  **Аналіз та створення шаблону**:
    *   Запустіть веб-інтерфейс або скористайтеся CLI.
    *   Оберіть один із режимів порівняння (*Повний автопілот*, *Пакетний аналіз* або *Попарне порівняння*).
    *   Система порівняє файли, виявить текстові розбіжності та замінить їх на універсальні змінні `{{field_X}}`.
    *   **Результат кроку**: Буде створено файл шаблону (наприклад, `template_Договір.docx`) та конфігураційний Excel-файл (наприклад, `Auto_Config.xlsx`).
3.  **Редагування бази даних (Excel-конфігу)**:
    *   Відкрийте створений Excel-конфіг у вбудованому інтерактивному табличному редакторі (або у зовнішньому MS Excel).
    *   Ви побачите змінні в рядку 4 та значення зі старих документів у рядках нижче.
    *   Ви можете редагувати тексти, додавати/видаляти рядки (нових клієнтів), додавати чи видаляти стовпчики (змінні).
    *   *Унікальна фіча*: Перейменування назви колонки (змінної) у веб-таблиці автоматично оновить відповідні теги `{{...}}` у підключеному файлі шаблону Word або Excel зі збереженням форматування!
4.  **Індивідуальна перевірка та Live Preview**:
    *   Оберіть конкретний документ (рядок) у лівій навігаційній панелі додатку.
    *   В основній частині екрана з'явиться зручна форма для редагування змінних обраного клієнта та вікно **живого попереднього перегляду (Live HTML Preview)**.
    *   Будь-які зміни значень у формі миттєво відтворюються на прев'ю майбутнього документа.
5.  **Масова генерація**:
    *   У розділі генерації вкажіть вихідну папку та запустіть процес.
    *   Система автоматично згенерує окремі файли для кожного рядка таблиці з підставленими значеннями за вказаним шаблоном імені файлу (наприклад, `docs/Договір_{{field_1}}.docx`).

## 🎮 Режими роботи через консоль (CLI)

Скрипт автоматично розпізнає потрібний режим на основі переданих йому аргументів.

### 🔍 Частина 1: Створення шаблонів та конфігів (Аналіз)

Скрипт порівнює документи, виявляє місця, де текст відрізняється, замінює ці розбіжності на змінні `{{field_1}}`, `{{field_2}}` тощо, зберігаючи оригінальне форматування, та генерує Excel-таблицю налаштувань.

#### 1️⃣ Режим "Повний автопілот"
Аналізує всю папку, групує схожі файли за структурою, створює для кожної групи один шаблон і загальний мульти-конфіг `Auto_Config.xlsx`.
```bash
python _templates_machine_.py <назва_папки>
```
*Приклад:* `python _templates_machine_.py "Архів"`

#### 2️⃣ Режим "Пакетний аналіз"
Бере один файл-зразок і порівнює його з усіма схожими файлами в папці для пошуку змінних.
```bash
python _templates_machine_.py <зразок> <папка>
```
*Приклад:* `python _templates_machine_.py "було.docx" "Архів"`

#### 3️⃣ Режим "Попарне порівняння"
Швидко порівнює два конкретних файли та генерує шаблон з конфігом.
```bash
python _templates_machine_.py <файл_1> <файл_2>
```
*Приклад:* `python _templates_machine_.py "Договір_Київ.docx" "Договір_Львів.docx"`

---

### 🖨️ Частина 2: Масова генерація документів (Синтез)

Зчитує дані зі створеного Excel-конфігу та створює готові документи з підставленими значеннями.

```bash
python _templates_machine_.py <файл_конфігу.xlsx> [аркуш] [рядок] [папка_виходу]
```

#### Варіанти фільтрації при запуску:
*   **Згенерувати все з конфігу**:
    ```bash
    python _templates_machine_.py "Auto_Config.xlsx"
    ```
*   **Генерувати тільки один конкретний аркуш**:
    ```bash
    python _templates_machine_.py "Auto_Config.xlsx" "Договори"
    ```
*   **Генерувати конкретний рядок з певного аркуша (наприклад, рядок 5)**:
    ```bash
    python _templates_machine_.py "Auto_Config.xlsx" "Договори" 5
    ```

## 🖥️ Графічний веб-інтерфейс (Streamlit Web GUI)

Запуск веб-інтерфейсу виконується командою:
```bash
streamlit run app.py
```

Додаток має сучасний двопанельний інтерфейс із адаптивним вмістом:

### 1️⃣ Ліва панель (Навігація по проекту)
*   **Кнопка `✈️ Створення конфігів`**: Перемикає головну область у режим аналізу папок.
*   **Кнопка `🌐 Усі змінні`**: Відкриває глобальний матричний аналізатор змінних по всьому проекту.
*   **Кнопка `📁 Папка конфігів`**: Дозволяє змінити поточну робочу директорію.
*   **Перемикач `Конфіги і шаблони` / `Віртуальні документи`**: Змінює режим відображення дерева (фізичні файли конфігів та шаблонів або віртуально згенерований список кінцевих документів).
*   **Інтерактивне дерево**:
    *   Папки відображаються у вигляді експандерів.
    *   Конфігураційні файли (позначені 📊) містять кнопку **`⚙️ Конфіг`** для переходу до табличного редактора.
    *   Аркуші конфігів містять списки документів та кнопку **`📝 Шаблон`** для перегляду структури плейсхолдерів.
    *   Клік на документ `📄 {doc_name}` відкриває його форму редагування та Live Preview.

### 2️⃣ Права панель (Основна робоча область)
Вміст цієї панелі динамічно змінюється залежно від вибору в лівому меню:
*   **Якщо обрано Папку (Аналіз)**: Налаштування та запуск режимів аналізу (Автопілот, Пакетний, Попарний), вибір вихідного каталогу та виведення логів виконання в реальному часі.
*   **Якщо обрано кнопку `⚙️ Конфіг`**: Інтерактивна таблиця `st.data_editor` для редагування значень, додавання/видалення рядків та колонок. Також тут розміщено блок метаданих (шлях до шаблону в комірці A1, шаблон імені в A2) та інтерфейс масової генерації документів.
*   **Якщо обрано документ `📄 {doc_name}`**: 
    *   Зручна форма для редагування змінних обраного документа.
    *   **Живий попередній перегляд (Live HTML Preview)** згенерованого документа в реальному часі.
    *   Кнопка миттєвого збереження змін в Excel-файл та кнопка завантаження згенерованого файлу.
*   **Якщо обрано кнопку `📝 Шаблон`**:
    *   Список знайдених плейсхолдерів у файлі шаблону та точний контекст їх використання (номер абзацу, таблиця, колонтитул тощо).
    *   Можливість відкрити файл шаблону у зовнішній програмі.
*   **Якщо обрано кнопку `🌐 Усі змінні`**:
    *   Двоколонковий зведений аналізатор: ліворуч — список унікальних змінних проекту, праворуч — таблиця їх значень по всіх конфігах та аркушах.
    *   **Крос-пошук**: Клік по будь-якому значенню запускає глобальний пошук документів у всьому проекті, що містять це значення.
    *   **Глобальний редактор змінних**: Можливість змінювати значення чи перейменовувати самі змінні по всьому проекту з інтелектуальним вирішенням конфліктів дублювання (злиття та видалення старої змінної або скасування).

## 💎 Унікальні переваги та інтелектуальні функції

### 🛡️ Авто-лікування кирилиці у Windows CMD (Anti-Mojibake)
Під час запуску скрипту через командний рядок Windows кириличні аргументи (наприклад, назви аркушів `"Аркуш1"`) часто перетворюються на нечитабельний набір символів. Функція [heal_cmd_string](file:///C:/Users/Usr/.gemini/antigravity/worktrees/Wodr-Excel-templates/analyze-code-structure/core/cmd_helpers.py#L7) автоматично детектує такі викривлення та відновлює оригінальний текст.

### 🎯 Розумне збереження пробілів та форматування
Пробіли на межах змінних (особливо при підкресленнях у Word) зберігаються до останнього символу. Вирівнювання, курсив, жирний шрифт та межі підкреслень залишаються рівно такими, якими були до автоматичної заміни.

### 🧩 Безпечна робота зі злитими комірками таблиць
Скрипт містить XML-трекінг комірок (`cell._tc`). Він захищає від помилок `NoneType` при аналізі об'єднаних зон Excel та виключає повторне сканування злитих комірок.

### 📂 Відтворення оригінальної структури папок (Змінна `rel_dir`)
Якщо оригінальні файли були вкладені в підпапки, система автоматично врахує це, пропише шлях у конфігурацію в колонку `rel_dir`, а під час генерації сама створить відсутні підпапки і розкладе готові документи на свои місця.

### 🛣️ Безпечна нормалізація відносних шляхів (`clean_relative_path`)
Усі відносні шляхи перед використанням проходять автоматичне очищення: видаляються початкові слеші (запобігає трактуванню шляхів як абсолютних на Windows), подвійні та зворотні слеші нормалізуються до єдиного стандарту (`/`).

### 🔄 Фонова синхронізація (Час модифікації `mtime`)
Система відстежує час останньої модифікації файлів Excel на диску (`mtime`). Будь-які зміни (навіть внесені вручну у зовнішньому MS Excel) миттєво оновлюють кеш додатку, прев'ю та дерево навігації без перемальовування всієї сторінки.

### 🎨 Динамічне стилювання в Excel
Проект підтримує динамічне стилювання згенерованих Excel-файлів за допомогою ручного додавання колонок із суфіксами стилю (наприклад, `field_1_bold`, `field_1_fill`, `field_1_border`, `field_1_number_format`). Модуль [xlsx_handler.py](file:///C:/Users/Usr/.gemini/antigravity/worktrees/Wodr-Excel-templates/analyze-code-structure/core/file_handlers/xlsx_handler.py) зчитує та десеріалізує ці значення через [excel_styles.py](file:///C:/Users/Usr/.gemini/antigravity/worktrees/Wodr-Excel-templates/analyze-code-structure/core/excel_styles.py), примусово форматуючи вихідні комірки.
