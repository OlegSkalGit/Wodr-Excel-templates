# 🚀 ІНТЕЛЕКТУАЛЬНА СИСТЕМА АВТОМАТИЗАЦІЇ ДОКУМЕНТІВ: _templates_machine_

Ласкаво просимо до **_templates_machine_** — універсального та розумного "комбайну" для автоматизації роботи з документами **Word (.docx)** та **Excel (.xlsx)**.

Система дозволяє повністю автоматизувати життєвий цикл документів: від аналізу архіву заповнених договорів та створення розумних шаблонів із виділенням змінних, до масової генерації сотень готових файлів на основі Excel-таблиць та їх інтерактивного редагування в реальному часі.

---

## 🚀 ШВИДКИЙ СТАРТ (Запуск за 3 кроки)

Для того щоб запустити систему на вашому комп'ютері, виконайте три прості кроки:

1. **Запустіть файл `_START_.bat`** у корені проекту.
   * Скрипт автоматично перевірить наявність Python, створить ізольоване віртуальне оточення `.venv` та самостійно встановить усі необхідні бібліотеки (`streamlit`, `pandas`, `openpyxl`, `docxtpl` тощо).
2. **Оберіть робочу папку (Workspace)** у веб-інтерфейсі, що відкриється в браузері.
   * Це папка, де будуть зберігатися ваші вихідні документи, шаблони та конфігурації.
3. **Почніть роботу** через зручний веб-інтерфейс!

## 🎯 ЯК ЦЕ ПРАЦЮЄ: ПОКРОКОВИЙ ЖИТТЄВИЙ ЦИКЛ РОБОТИ

Система об'єднує два головних процеси: **Аналіз** (створення шаблонів з ваших файлів) та **Синтез** (генерація нових документів).

### Крок 1: Аналіз та створення шаблону
У вас є 3 договори різних клієнтів із різними іменами та сумами. 
* Ви завантажуєте їх у систему. 
* Програма автоматично знаходить розбіжності (імена, суми, адреси) і замінює їх на універсальні теги: `{{field_1}}`, `{{field_2}}` тощо.
* **Результат**: Ви отримуєте чистий файл-шаблон (`template_Договір.docx`) та Excel-файл налаштувань (`config_Auto.xlsx`), де у рядках записані дані ваших клієнтів, а в колонках — змінні.

### Крок 2: Редагування бази даних (Excel-конфігу)
* Ви відкриваєте створений Excel-конфіг безпосередньо у вбудованому веб-редакторі таблиць.
* Ви можете додавати нових клієнтів (рядки), редагувати їхні дані або дописувати формули.

### Крок 3: Масова генерація документів
* Натисканням однієї кнопки система зчитує дані з Excel-таблиці, підставляє їх у шаблон та генерує готові Word або Excel файли для кожного клієнта.
* Імена файлів створюються автоматично за вашим шаблоном (наприклад, `Договір_Київ_Скалацький.docx`).

## 🖥️ МОЖЛИВОСТІ ВЕБ-ІНТЕРФЕЙСУ (Streamlit Web GUI)

Графічний інтерфейс побудований за зручною трипанельною структурою:

### 1️⃣ Ліва панель: Навігація по проекту
* **Інтерактивне дерево файлів**: Відображає папки, конфігурації Excel (позначені 📊), аркуші конфігів, шаблони (позначені 📝) та окремі документи (📄).
* **Режими дерева**:
  * *Конфіги і шаблони*: Відображає фізичні файли на диску.
  * *Віртуальні документи*: Перетворює рядки Excel-таблиць на віртуальний список кінцевих документів (клік по документу одразу відкриває його форму).
* **Швидкі кнопки**:
  * `✈️ Створення конфігів`: Перехід у режим аналізу та генерації нових шаблонів.
  * `🌐 Усі змінні`: Глобальний аналізатор усіх тегів у проекті.
  * `📁 Папка конфігів`: Зміна поточного робочого каталогу (Workspace).

### 2️⃣ Режими аналізу (Створення шаблонів)
Коли ліворуч обрано робочу папку, права панель дозволяє запускати один із трьох режимів розпізнавання:
* **Повний автопілот**: Сканує обрану папку, групує схожі за структурою документи, створює для кожної групи окремий шаблон та зводить усі дані в один файл `config_Auto.xlsx`.
  * *Ігнорувати одиничні файли*: Якщо активовано, одиничні файли (без дублікатів) не оброблятимуться взагалі. Якщо вимкнено — вони будуть винесені в окремий конфіг `config__NODublicate_.xlsx` та папку `_NODublicate_` у папці результатів і додаватимуться як окремі аркуші в цьому конфігу.
* **Пакетний аналіз**: Ви вказуєте один файл-зразок та папку з іншими аналогічними файлами. Система порівнює їх та створює шаблон на основі зразка.
* **Попарне порівняння**: Порівнює два конкретних файли та створює шаблон із виявленими змінними.

### 3️⃣ Інтерактивний редактор таблиць Excel
Клік по кнопці `⚙️ Конфіг` біля будь-якого аркуша відкриває повноцінний табличний редактор:
* **Зміна значень на льоту**: Редагуйте дані в таблиці, додавайте нові рядки або видаляйте непотрібні.
* **Магічна синхронізація тегів (Magic Sync)**: Якщо ви перейменуєте назву стовпчика (наприклад, зміните `field_1` на `ПІБ_Клієнта`), система **автоматично оновить** усі відповідні теги `{{field_1}}` на `{{ПІБ_Клієнта}}` всередині підключеного файлу шаблону Word або Excel, зберігаючи оригінальне форматування та стилі!
* **Масова генерація**: Прямо під таблицею ви можете обрати масштаб генерації (поточний аркуш або всі аркуші проекту), обрати конкретний рядок або запустити генерацію для всіх даних.

### 4️⃣ Форма редагування та Живий прев'ю (Live HTML Preview)
Клік по конкретному документу (📄) відкриває індивідуальну робочу область:
* **Живий попередній перегляд (Live HTML Preview)**: Зліва відображається форма зі змінними обраного клієнта, а праворуч — інтерактивне вікно документа. Будь-які зміни у формі миттєво відтворюються на HTML-прев'ю майбутнього файлу (підтримує як Word, так і Excel).
* **Кнопка "Зберегти конфіг"**: Записує відредаговані значення назад в Excel.
* **Кнопка "Згенерувати файл"**: Завантажує готовий файл через браузер або зберігає у вказану папку на диску.

### 5️⃣ Аналізатор шаблонів
Клік по шаблону (📝) дозволяє:
* Переглянути повний перелік виявлених змінних у цьому шаблоні та точний контекст їх використання (номер абзацу, таблиця, колонтитул тощо).
* **Безпечне перейменування**: Перейменуйте змінну в шаблоні, і система автоматично оновить заголовки стовпчиків у пов'язаних конфігураціях Excel.
* Відкрити шаблон у зовнішньому додатку (MS Word / Excel) для ручного редагування дизайну.

### 6️⃣ Глобальний аналізатор змінних (`🌐 Усі змінні`)
* **Зведений реєстр**: Відображає всі унікальні змінні по всьому проекту.
* **Фільтрація за значенням**: Поле фільтру над таблицею «Змінні проекту» дозволяє миттєво відсіяти змінні за їхнім значенням у конфігах (без урахування регістру). У лівій таблиці фільтруються змінні (рядки), а в правій — конфіги (колонки), відображаючи тільки ті стовпці, де змінна набуває шуканого значення (інші рядки у цих колонках також залишаються видимими).
* **Крос-пошук**: Клік по будь-якому значенню запускає глобальний пошук і показує список усіх документів у проекті, де це значення зустрічається.
* **Глобальний редактор**: Дозволяє перейменувати змінну або змінити її значення одразу у всіх конфігураціях та шаблонах проекту.

## 🎮 РОБОТА ЧЕРЕЗ КОНСОЛЬ (CLI)

Ви можете запускати всі операції через командний рядок Windows. Скрипт автоматично розпізнає режим на основі переданих аргументів.

### 🔍 Режими аналізу та створення шаблонів
* **Повний автопілот**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "Шлях_до_папки_з_архівом" [--ignore-single]
  ```
  *Використання прапорця `--ignore-single` вказує системі повністю ігнорувати одиничні файли (без дублікатів) і не обробляти їх. Без цього прапорця вони автоматично виноситимуться в окремий конфіг `config__NODublicate_.xlsx` та папку `_NODublicate_`.*
* **Пакетний аналіз**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "зразок.docx" "папка_порівняння" [папка_результатів]
  ```
* **Попарне порівняння**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "договір1.docx" "договір2.docx" [папка_результатів]
  ```

### 🖨️ Режими генерації документів
Синтаксис: `_templates_machine_.py <конфіг.xlsx> [аркуш] [рядок] [папка_виходу]`
* **Згенерувати все з конфігу**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "config_Auto.xlsx"
  ```
* **Генерувати тільки один аркуш (наприклад, "Договори")**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "config_Auto.xlsx" "Договори"
  ```
* **Генерувати один рядок (наприклад, рядок 5)**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "config_Auto.xlsx" "Договори" 5
  ```
* **Зберегти результати у певну папку**:
  ```bash
  .venv\Scripts\python _templates_machine_.py "config_Auto.xlsx" "Договори" all "C:\GeneratedDocs"
  ```

## 💎 ПРОСУНУТІ МОЖЛИВОСТІ ТА ІНТЕЛЕКТУАЛЬНІ ФУНКЦІЇ

* **Динамічне стилювання в Excel**: Ви можете керувати оформленням згенерованих Excel-файлів, додаючи в таблицю спеціальні стовпчики-суфікси стилів. Наприклад, якщо у вас є колонка `Сума`, створення колонок `Сума_bold` (зі значенням `True`) або `Сума_fill` (з кодом кольору, наприклад `FF0000`) автоматично зробить шрифт жирним або зафарбує клітинку у підсумковому документі.
* **Збереження оригінального форматування**: При заміні тегів `{{змінна}}` у Word система зберігає всі налаштування шрифтів, підкреслень, вирівнювання, курсиву та кольору тексту, які були задані в шаблоні.
* **Авто-лікування кодування в консолі**: У Windows CMD кириличні аргументи часто спотворюються. Система містить захисний механізм, який автоматично розпізнає та виправляє нечитабельні символи.
* **Підтримка злитих комірок Excel**: Система безпечно обробляє об'єднані зони в Excel-шаблонах, запобігаючи виникненню помилок `NoneType` при копіюванні стилів та даних.
* **Фонова синхронізація з диском**: Якщо ви відкриєте конфігураційний Excel-файл у звичайному MS Excel та внесете туди зміни, веб-додаток миттєво помітить оновлення файлу на диску та актуалізує прев'ю та дерево файлів без перезавантаження сторінки.
* **Відтворення відносної структури папок**: Завдяки колонці `rel_dir` у конфігу, генератор документів може автоматично відтворювати структуру підкаталогів у вихідній папці та розкладати готові файли по відповідних теках.
